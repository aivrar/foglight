# Coverage Policy

Phase 0 records the legacy server at 58% combined line/branch coverage from 68
Python tests. `coverage.xml` and the terminal missing-lines report identify the
remaining gaps, concentrated in live network/cache failure permutations,
desktop lifecycle, and the monolithic HTTP route dispatcher. The legacy number
is diagnostic and is not raised cosmetically with exclusions.

The current browser script is not an ES module, so Node cannot import and
instrument it safely without executing the application. Its behavior is guarded
by deterministic Playwright interactions and screenshots until Phase 1 extracts
pure modules. The JavaScript unit command already emits branch coverage for
each importable module it exercises.

Release floors for new code are:

- At least 90% branch coverage for `foglight_core` and extracted pure browser
  modules.
- 100% of documented rule branches for scoring, correlation, migrations,
  retention, and security-boundary rules.
- Every provider parser must cover valid, empty, malformed, and upstream-error
  behavior.
- Coverage may fall only with an explicit, reviewed rationale tied to
  unreachable platform behavior; blanket file exclusions are not accepted.

Coverage is generated by `python -m pytest` and
`npm run test:js`. The browser suite remains an independent integration gate.
